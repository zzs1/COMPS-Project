Local Storage Explained

When you are designing or trying to make anything you may hit a roadblock if you are not a back end developer or you do not want to go down the complicated path to be able to store certain data. This is where localStorage comes into play. localStorage is basically a way to store data locally on a user's browser that stays there whether you exit the page, reload the browser or change pages. The way it works is that there are keys and values. For example, you could say "name: Jacob", the name is considered the key and the value of name is Jacob. To make use of these values there are 4 main methods,
1. setItem() - Makes a key and value then automatically adds it to localStorage until removed or cleared
2. getItem() - Gets the key of whatever you input and returns the value it has
3. removeItem() - Gets the key of whatever you input and removes both the key and the value attached
4. clear() - Clears the localStorage
